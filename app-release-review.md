# Yorimichi（ヨリミチ）プロフェッショナルレビュー

## アプリ概要
**位置情報ベースの買い物リマインダーアプリ**（React Native + Android ネイティブ）
ユーザーが買い物メモに店舗の位置を登録すると、ジオフェンスでその店舗に近づいた際にプッシュ通知でリマインドする。

---

## 1. リリース判定：このまま進んで問題ないか

### 合格点（リリース可能な水準にある項目）

- **コアUX完成度**: メモ作成 → 場所登録 → ジオフェンス通知 の基本フローが完成
- **ネイティブジオフェンス実装**: Google Play Services `GeofencingClient` を使用しており、バッテリー効率が高い。アプリ killed 状態でも通知が届く
- **状態管理**: Zustand + MMKV（暗号化ストレージ）による堅実な設計
- **マイグレーション対応**: ストアバージョン管理（v1→v9）が適切に実装済み
- **i18n対応**: 日英2言語対応
- **クラッシュレポート**: Firebase Crashlytics 導入済み
- **ProGuard/R8**: リリースビルドで有効化、主要ライブラリの keep ルールも設定済み
- **テスト**: 10個のテストファイルが存在（ストア、サービス、ユーティリティをカバー）

### 要改善（リリース前に対応推奨）

| # | 項目 | 重要度 | 詳細 |
|---|------|--------|------|
| 1 | **権限リクエストUIが日本語ハードコード** | 中 | `App.tsx:70-76` で位置情報の説明ダイアログが日本語固定。i18n の翻訳キーを使うべき |
| 2 | **プライバシーポリシー未確認** | 高 | Google Play ストア掲載には位置情報・匿名認証・広告に関するプライバシーポリシーが必須。コードベースにリンクが見当たらない |
| 3 | **CI/CD未構築** | 中 | GitHub Actions 等が未設定。手動ビルド・手動デプロイはリリース後のメンテで事故リスクあり |
| 4 | **iOS未対応** | 低 | iOS 側のコードはあるがジオフェンスが Android only。意図的であれば問題なし |
| 5 | **`comingSoon` 機能のハードコード日付** | 中 | `PremiumScreen.tsx:37` に「4月」とハードコードされた coming soon バッジがある。リリース時期に合わせて更新が必要 |

## 2. セキュリティ評価

### 良好な点

| 項目 | 評価 |
|------|------|
| **APIキー管理** | `.env` + `react-native-config` で管理。`.env` ファイルは git 追跡外 |
| **Firestore セキュリティルール** | UID ベースのアクセス制御、フィールド検証、サイズ制限あり。クーポンは `used: false→true` のみ許可 |
| **匿名認証** | Firebase Anonymous Auth で認証。競合防止のシングルトンパターンも適切 |
| **ローカルストレージ** | MMKV（暗号化対応）を使用 |
| **keystore** | `keystore.properties` は git 追跡外、`.example` ファイルのみコミット |
| **google-services.json** | git 追跡外 |
| **ownerUid 書き換え防止** | Firestore ルールで `ownerUid` の変更を明示的にブロック |
| **ProGuard 有効** | リリースビルドでコード難読化が有効 |

### 要注意点

| # | 項目 | 深刻度 | 詳細 |
|---|------|--------|------|
| 1 | **トライアル期間のクライアントサイド制御** | 中 | `trialStartDate` が MMKV に保存されており、端末の日時変更やアプリ再インストールで回避可能。ただしトライアルは集客施策なので致命的ではない |
| 2 | **`isLegacyDoc()` 移行ルール** | 中 | `firestore.rules:12-13` で `ownerUid` 未設定のドキュメントは認証済みユーザー全員にアクセスを許可。移行完了後にこのルールを削除すべき |
| 3 | **PendingIntent `FLAG_MUTABLE`** | 低 | `GeofenceModule.java:68` で `FLAG_MUTABLE` を使用。ジオフェンス用途では必要だが、他の箇所では `FLAG_IMMUTABLE` を使用（`GeofenceTransitionReceiver.java:128`）しており適切 |
| 4 | **SharedPreferences MODE_PRIVATE 未指定** | 低 | `GeofenceModule.java:89-91` で `getSharedPreferences(PREFS_NAME, 0)` を使用。`0` は `MODE_PRIVATE` と同値なので実害なし |
| 5 | **バックアップの deviceId キー** | 低 | バックアップドキュメントの ID が `deviceId`（UUID）で、`ownerUid` とは別。Firestore ルールで `ownerUid` 一致を検証しているため問題なし |
| 6 | **クーポンの brute-force 対策** | 低 | クーポンコードは Firestore の doc ID でルックアップ。レートリミットがないため大量試行は可能だが、Firestore の読み取りコストが抑止力になる |

### セキュリティ総合評価: **概ね良好**
個人開発アプリとしては十分なセキュリティ水準。致命的な脆弱性は見つからなかった。

## 3. 収益化評価

### 現在のモデル

```
収益源1: Google AdMob（バナー + インタースティシャル）→ 無料ユーザー向け
収益源2: RevenueCat サブスクリプション（月額 + 年額）→ プレミアム
```

### 無料 → プレミアム の差別化

| 項目 | 無料 | プレミアム |
|------|------|-----------|
| メモ数 | 5 | 1000 |
| アイテム数/メモ | 10 | 100 |
| 登録地点/メモ | 2 | 10 |
| 共有相手 | 1人 | 20人 |
| リアルタイム同期 | - | あり |
| クラウドバックアップ | - | あり |
| 広告 | あり | なし |
| 通知時間帯設定 | - | あり |

### 収益化の強み
- **7日間トライアル**: プレミアム体験後のコンバージョン導線がある
- **クーポンシステム**: プロモーション・インフルエンサー施策に活用可能
- **広告 + サブスクのハイブリッド**: 無料ユーザーからも収益を得られる構造
- **年額プランに「Best Value」バッジ**: ARPU 向上の工夫

### 収益化の課題・改善提案

| # | 提案 | 期待効果 |
|---|------|----------|
| 1 | **無料メモ数を3に削減** | planLimits.ts のコメントにも記載あり。5→3 でプレミアム転換率向上の可能性。ただしリテンション低下リスクとのA/Bテストが必要 |
| 2 | **トライアル終了前のプッシュ通知** | トライアル残り3日・1日時点でリマインド通知を送り、転換率を向上 |
| 3 | **インタースティシャル広告のタイミング拡大** | 現在はメモ登録時のみ。地点追加時にも表示すれば広告インプレッション増加 |
| 4 | **RevenueCat Paywall テンプレート** | A/Bテスト機能を活用してプレミアム画面のCVRを最適化 |
| 5 | **Onboarding での価値訴求** | 初回起動時にジオフェンス通知のデモを見せ、プレミアムの「場所10件」の価値を実感させる |
| 6 | **リファラルプログラム** | 共有機能を活かし、友人招待でクーポン付与する仕組み |

### 収益化総合評価: **収益化の骨格は整っている**
マネタイズの仕組みは一通り揃っている。あとは無料制限の調整とコンバージョン導線の最適化がカギ。

## 4. コード品質・アーキテクチャ評価

### 良い点
- **関心の分離**: screens / services / store / hooks / utils が明確に分離
- **ネイティブブリッジ設計**: ジオフェンスをネイティブ実装し、JS側はオーケストレーションに専念
- **Firestore undefined 対策**: `sanitizeItem()` / `sanitizeLocation()` で JSON round-trip の罠を回避
- **エラーハンドリング**: Crashlytics への `recordError` が要所で使われている
- **型安全性**: TypeScript の型定義が適切
- **ストア設計**: Zustand の persist + migrate パターンが洗練されている

### 改善点

| # | 項目 | 詳細 | ファイル |
|---|------|------|----------|
| 1 | **PremiumScreen.tsx が 816行** | 比較テーブル、購入ロジック、バックアップ、クーポンが1ファイルに集中。コンポーネント分割を推奨 | `src/screens/PremiumScreen.tsx` |
| 2 | **BootReceiver の実装が空** | `BootReceiver.kt` が端末再起動後のジオフェンス再登録に使われているはずだが、実装内容を確認すべき | `android/.../BootReceiver.kt` |
| 3 | **`syncAllSharedMemos` の N+1 問題** | `Promise.all` で各 doc を個別取得。Firestore の `getAll` バッチ読み取りを使うべき | `src/services/shareService.ts:131` |
| 4 | **広告表示のプレミアム判定がクライアントのみ** | `selectEffectivePremium` がローカルストアの値に依存。サーバーサイド検証なし（RevenueCat Webhook でサーバー連携すると堅牢化できる） |
| 5 | **E2Eテスト未実装** | ユニットテストはあるが、Detox/Maestro 等のE2Eテストがない |
